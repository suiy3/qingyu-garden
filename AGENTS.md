# 晴语项目交接说明

## 项目定位

晴语是一款面向初高中学生的移动端学习与情绪记录应用。核心体验是用约一分钟完成“心情 + 学习”双轨记录，再通过趋势、知识点、错题和微行动帮助学生逐步改善状态。

## 接手时必须保留的产品原则

- 学生优先，文字语气温和、克制，不幼稚，也不制造焦虑。
- 数据默认仅保存在当前浏览器的 `localStorage`，不要在未获用户明确同意时改成上传云端。
- 家长视图只能查看概括性趋势，不能显示学生记录原话、知识笔记或聊天内容。
- 家长入口需要二次确认，并在 30 分钟后自动退出。
- “智能发现”是基于本地数据的规律提示，不是医学或心理诊断。
- 保留演示数据模式；用户第一次创建真实记录后，演示数据应退出，避免混入个人统计。
- 不要在未获用户明确同意时覆盖或推送到原有的公开比赛仓库。

## 当前主要功能

- `/`：首页与知识花园
- `/check-in`：每日“心情 + 学习”合并打卡
- `/knowledge`：知识点与错题列表
- `/knowledge/new`：新建知识点或错题
- `/insight`：本地规律发现
- `/actions`：微行动
- `/weekly-report`：周报
- `/profile`：个人、隐私与数据管理
- `/parent`：家长趋势视图

## 技术与运行方式

- React 18 + TypeScript + Vite + Tailwind CSS + Zustand + React Router
- 推荐 Node.js 18 或更高版本
- 安装依赖：`npm install`
- 本地开发：`npm run dev`
- 常规构建：`npm run build`
- ChatGPT Sites 构建（PowerShell）：`$env:SITES_BUILD='true'; npm run build`
- GitHub Pages 的基础路径仍为 `/qingyu-garden/`，不要随意删除相关配置。

## 修改建议

修改前先通读 `src/store/useAppStore.ts`、`src/App.tsx` 以及目标页面。完成后至少运行 TypeScript 检查和生产构建，并确认移动端首页、打卡、知识点/错题、个人页及家长页没有回归问题。
